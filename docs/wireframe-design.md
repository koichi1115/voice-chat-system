# ワイヤーフレーム設計

## 設計原則

### 子供向けUIの基本原則
1. **シンプル・直感的**: 最小限の要素で最大の効果
2. **大きなタッチターゲット**: 指でも操作しやすいサイズ
3. **明確な視覚的フィードバック**: 操作結果がすぐに分かる
4. **エラー許容設計**: 間違いを許容し、簡単に回復できる

### レイアウト基本方針
- **縦方向中心**: 重要な要素を縦に配置
- **中央配置**: 主要コンテンツをスクリーン中央に
- **余白を活用**: 要素間に十分なスペースを確保
- **一画面完結**: スクロールを最小限に抑制

## インタラクション設計

### メインボタンの動作
1. **通常状態**: 「🎤 話しかける」
2. **ホバー/フォーカス**: ボタンが光る・拡大
3. **押下**: 「🔴 録音中」に変化
4. **音声認識中**: 波形アニメーション表示
5. **完了**: 「🤖 考えています...」

### 状態遷移フロー
```
[待機] → [音声入力] → [音声認識] → [AI処理] → [音声出力] → [完了] → [待機]
   ↑                                                            ↓
   └─────────────── [エラー時の復帰] ←─────────────────────────┘
```

### フィードバック要素
- **視覚**: 色の変化、アニメーション、アイコン変更
- **音声**: ボタンクリック音、録音開始音、完了音
- **テキスト**: 状態の文字による説明

## アクセシビリティ配慮

### キーボード操作
- Tabキーで主要要素間移動
- Enterキーでボタン実行
- Escapeキーで操作キャンセル

### スクリーンリーダー対応
- 適切なaria-label設定
- roleの明確な定義
- 状態変化の音声通知

### 視覚的配慮
- 高コントラスト色の使用
- 十分な文字サイズ（最小16px）
- 色だけに依存しない情報伝達

## レスポンシブ対応

### ブレークポイント
- **デスクトップ**: 1024px以上
- **タブレット**: 768px-1023px  
- **モバイル**: 767px以下

### 画面サイズ別調整
- **デスクトップ**: 中央配置、サイドバー表示
- **タブレット**: 縦レイアウト、フルスクリーン
- **モバイル**: 最適化UI、大きなタッチターゲット
